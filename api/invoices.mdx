## Overview

The Invoices API allows you to manage invoices programmatically. Invoices represent sales documents issued to customers, with support for payments, taxes, discounts, and multi-currency.

## Invoice Object

The invoice object contains the following fields:

<ResponseField name="id" type="integer">
  Unique identifier for the invoice
</ResponseField>

<ResponseField name="workspace_id" type="integer">
  ID of the workspace this invoice belongs to
</ResponseField>

<ResponseField name="contact_id" type="integer">
  ID of the customer contact
</ResponseField>

<ResponseField name="document_subtype_id" type="integer">
  ID of the document subtype (determines NCF format for Dominican Republic)
</ResponseField>

<ResponseField name="document_number" type="string">
  Invoice number (e.g., "INV-0001", "B0100000001")
</ResponseField>

<ResponseField name="issue_date" type="date">
  Date the invoice was issued (YYYY-MM-DD)
</ResponseField>

<ResponseField name="due_date" type="date">
  Payment due date (YYYY-MM-DD)
</ResponseField>

<ResponseField name="status" type="enum">
  Invoice status: `pending_payment`, `partially_paid`, `paid`, `cancelled`, `draft`
</ResponseField>

<ResponseField name="subtotal_amount" type="decimal">
  Subtotal before taxes and discounts
</ResponseField>

<ResponseField name="tax_amount" type="decimal">
  Total tax amount
</ResponseField>

<ResponseField name="discount_amount" type="decimal">
  Total discount amount
</ResponseField>

<ResponseField name="total_amount" type="decimal">
  Final total amount
</ResponseField>

<ResponseField name="amount_due" type="decimal">
  Outstanding balance (total_amount - amount_paid)
</ResponseField>

<ResponseField name="amount_paid" type="decimal">
  Total amount paid toward this invoice
</ResponseField>

<ResponseField name="currency_id" type="integer">
  Currency for this invoice
</ResponseField>

<ResponseField name="payment_term" type="string">
  Payment terms (e.g., "Net 30", "Due on Receipt")
</ResponseField>

<ResponseField name="notes" type="string">
  Additional notes or terms and conditions
</ResponseField>

<ResponseField name="items" type="array">
  Array of invoice line items
</ResponseField>

<ResponseField name="payments" type="array">
  Array of payments applied to this invoice
</ResponseField>

<ResponseField name="salesmen" type="array">
  Array of salesmen associated with this invoice
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  Creation timestamp
</ResponseField>

<ResponseField name="updated_at" type="timestamp">
  Last update timestamp
</ResponseField>

## List Invoices

<api>GET /invoices</api>

Retrieve a paginated list of invoices.

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="per_page" type="integer" default="15">
  Number of items per page (max 100)
</ParamField>

<ParamField query="filter[status]" type="string">
  Filter by invoice status: `pending_payment`, `paid`, `partially_paid`, `cancelled`
</ParamField>

<ParamField query="filter[contact_id]" type="integer">
  Filter by customer contact ID
</ParamField>

<ParamField query="filter[issue_date]" type="date">
  Filter by issue date (YYYY-MM-DD)
</ParamField>

<ParamField query="sort" type="string" default="-issue_date">
  Sort field (prefix with `-` for descending). Options: `issue_date`, `document_number`, `total_amount`
</ParamField>

### Response

<CodeGroup>

```json 200 OK
{
  "data": [
    {
      "id": 1,
      "workspace_id": 2,
      "contact_id": 45,
      "document_subtype_id": 3,
      "document_number": "B0100000001",
      "issue_date": "2024-03-15",
      "due_date": "2024-04-14",
      "status": "pending_payment",
      "subtotal_amount": "1250.00",
      "tax_amount": "225.00",
      "discount_amount": "0.00",
      "total_amount": "1475.00",
      "amount_due": "1475.00",
      "amount_paid": "0.00",
      "currency_id": 1,
      "payment_term": "Net 30",
      "notes": "Terms and conditions apply",
      "created_at": "2024-03-15T10:30:00Z",
      "updated_at": "2024-03-15T10:30:00Z"
    }
  ],
  "meta": {
    "current_page": 1,
    "last_page": 3,
    "per_page": 15,
    "total": 42
  }
}
```

</CodeGroup>

### Example

<CodeGroup>

```bash cURL
curl -X GET "https://acme.yourdomain.com/api/v1/invoices?filter[status]=pending_payment&sort=-issue_date" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/json"
```

```php PHP
$response = Http::withToken($token)
    ->get('https://acme.yourdomain.com/api/v1/invoices', [
        'filter' => ['status' => 'pending_payment'],
        'sort' => '-issue_date'
    ]);

$invoices = $response->json('data');
```

```javascript JavaScript
const response = await fetch(
  'https://acme.yourdomain.com/api/v1/invoices?filter[status]=pending_payment&sort=-issue_date',
  {
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN',
      'Accept': 'application/json'
    }
  }
);

const { data: invoices } = await response.json();
```

</CodeGroup>

## Create Invoice

<api>POST /invoices</api>

Create a new invoice.

### Request Body

<ParamField body="contact_id" type="integer" required>
  Customer contact ID
</ParamField>

<ParamField body="document_subtype_id" type="integer" required>
  Document subtype ID (determines NCF sequence)
</ParamField>

<ParamField body="issue_date" type="date" required>
  Invoice issue date (YYYY-MM-DD)
</ParamField>

<ParamField body="due_date" type="date">
  Payment due date (YYYY-MM-DD)
</ParamField>

<ParamField body="payment_term" type="string">
  Payment terms (e.g., "Net 30")
</ParamField>

<ParamField body="notes" type="string">
  Additional notes or terms
</ParamField>

<ParamField body="currency_id" type="integer">
  Currency ID (defaults to workspace default currency)
</ParamField>

<ParamField body="items" type="array" required>
  Array of invoice line items
</ParamField>

<ParamField body="items[].product_id" type="integer" required>
  Product ID for this line item
</ParamField>

<ParamField body="items[].description" type="string">
  Line item description (defaults to product name)
</ParamField>

<ParamField body="items[].quantity" type="decimal" required>
  Quantity
</ParamField>

<ParamField body="items[].unit_price" type="decimal" required>
  Unit price
</ParamField>

<ParamField body="items[].discount" type="decimal" default="0">
  Discount amount or percentage
</ParamField>

<ParamField body="items[].tax_ids" type="array">
  Array of tax IDs to apply to this item
</ParamField>

<ParamField body="salesman_ids" type="array">
  Array of salesman IDs associated with this invoice
</ParamField>

### Response

<CodeGroup>

```json 201 Created
{
  "data": {
    "id": 42,
    "workspace_id": 2,
    "contact_id": 45,
    "document_subtype_id": 3,
    "document_number": "B0100000042",
    "issue_date": "2024-03-20",
    "due_date": "2024-04-19",
    "status": "pending_payment",
    "total_amount": "1475.00",
    "created_at": "2024-03-20T14:30:00Z"
  },
  "message": "Factura creada exitosamente."
}
```

```json 422 Validation Error
{
  "message": "The given data was invalid.",
  "errors": {
    "contact_id": [
      "The contact id field is required."
    ],
    "items": [
      "The items field must have at least 1 item."
    ]
  }
}
```

</CodeGroup>

### Example

<CodeGroup>

```bash cURL
curl -X POST https://acme.yourdomain.com/api/v1/invoices \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "contact_id": 45,
    "document_subtype_id": 3,
    "issue_date": "2024-03-20",
    "due_date": "2024-04-19",
    "payment_term": "Net 30",
    "notes": "Terms and conditions apply",
    "items": [
      {
        "product_id": 78,
        "description": "Premium Product",
        "quantity": 10,
        "unit_price": 125.00,
        "discount": 0,
        "tax_ids": [1]
      }
    ],
    "salesman_ids": [5]
  }'
```

```php PHP
$invoice = Http::withToken($token)
    ->post('https://acme.yourdomain.com/api/v1/invoices', [
        'contact_id' => 45,
        'document_subtype_id' => 3,
        'issue_date' => '2024-03-20',
        'due_date' => '2024-04-19',
        'payment_term' => 'Net 30',
        'notes' => 'Terms and conditions apply',
        'items' => [
            [
                'product_id' => 78,
                'description' => 'Premium Product',
                'quantity' => 10,
                'unit_price' => 125.00,
                'tax_ids' => [1],
            ],
        ],
        'salesman_ids' => [5],
    ])
    ->json('data');
```

```javascript JavaScript
const { data: invoice } = await fetch(
  'https://acme.yourdomain.com/api/v1/invoices',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN',
      'Content-Type': 'application/json',
      'Accept': 'application/json'
    },
    body: JSON.stringify({
      contact_id: 45,
      document_subtype_id: 3,
      issue_date: '2024-03-20',
      due_date: '2024-04-19',
      payment_term: 'Net 30',
      items: [
        {
          product_id: 78,
          quantity: 10,
          unit_price: 125.00,
          tax_ids: [1]
        }
      ],
      salesman_ids: [5]
    })
  }
).then(r => r.json());
```

</CodeGroup>

<Note>
  Creating an invoice automatically generates stock movements if products track inventory. The document number is auto-generated based on the document subtype's NCF sequence.
</Note>

## Update Invoice

<api>PATCH /invoices/{id}</api>

Update an existing invoice.

<Warning>
  Invoices can only be updated if their status is `pending_payment`. Invoices that are `paid`, `partially_paid`, or `cancelled` cannot be edited.
</Warning>

### Path Parameters

<ParamField path="id" type="integer" required>
  Invoice ID
</ParamField>

### Request Body

Same as Create Invoice endpoint. All fields are optional.

### Response

<CodeGroup>

```json 200 OK
{
  "data": {
    "id": 42,
    "document_number": "B0100000042",
    "total_amount": "1650.00",
    "updated_at": "2024-03-20T15:45:00Z"
  },
  "message": "Factura actualizada exitosamente."
}
```

```json 403 Forbidden
{
  "message": "Esta factura tiene un pago registrado. Para editarla, primero elimina los pagos asociados."
}
```

</CodeGroup>

## Delete Invoice

<api>DELETE /invoices/{id}</api>

Delete an invoice.

<Warning>
  Only invoices with status `pending_payment` or `draft` can be deleted. Paid or partially paid invoices must have their payments removed first.
</Warning>

### Path Parameters

<ParamField path="id" type="integer" required>
  Invoice ID
</ParamField>

### Response

<CodeGroup>

```json 200 OK
{
  "message": "Factura eliminada exitosamente."
}
```

```json 403 Forbidden
{
  "message": "Cannot delete an invoice with payments."
}
```

</CodeGroup>

## Download Invoice PDF

<api>GET /invoices/{id}/pdf</api>

Download invoice as PDF document.

### Path Parameters

<ParamField path="id" type="integer" required>
  Invoice ID
</ParamField>

### Response

Returns a PDF file with `Content-Type: application/pdf`.

### Example

```bash
curl -X GET https://acme.yourdomain.com/api/v1/invoices/1/pdf \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o invoice-001.pdf
```

## Validate NCF

<api>POST /invoices/validate-ncf</api>

Validate NCF (Dominican tax receipt number) for an invoice.

### Request Body

<ParamField body="ncf" type="string" required>
  NCF number to validate
</ParamField>

<ParamField body="document_subtype_id" type="integer" required>
  Document subtype ID
</ParamField>

### Response

```json
{
  "valid": true,
  "message": "NCF is valid and available."
}
```
